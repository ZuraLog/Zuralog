Three modes — the thinking behind each

  • Open Strava Recommended
    Deep-links directly into the Strava app to start a run. When the run is done and saved in Strava, it syncs back to Zuralog automatically via the Strava integration. If Strava is not connected yet, this button takes the user to the Integrations settings screen to link their account. Zuralog gets all the rich data — distance, pace, heart rate, route, splits — without building any of that itself.
  • Log a past run — simple manual form. Activity type, distance, duration. Pace auto-calculated. Optional effort rating and notes. For treadmill sessions, runs done without a phone, or users who don't use Strava. Intentionally kept simple — no conditions chips, no factors. Just the essentials.
  • Record live session Coming Soon — in-app GPS recording with live stats. A significant future feature. Shown greyed out so users know it's coming.

Manual form — kept deliberately simple

  • Activity type — Run, Walk, Cycle, Swim, Hike, Other. Required.
  • Distance — km or miles. Required.
  • Duration — mm:ss. Required.
  • Pace — auto-calculated from distance ÷ duration. Never an input field. auto
  • Effort — Easy / Steady / Hard / Max. optional
  • Notes — free text. optional
  • That's it. No conditions, no splits, no heart rate. Those come in via Strava. This form is for the user who just wants to record that they ran 5km this morning.
The Strava relationship: When Strava is connected, runs logged there appear automatically in Zuralog's Today feed and feed into the Health Score and AI insights. The "Open Strava" button just makes the flow explicit — tap here, go record your run, come back and it's already here. No duplicate entry, no friction.
