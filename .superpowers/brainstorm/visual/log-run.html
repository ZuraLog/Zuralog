Run / Cardio — Field Breakdown

  • Activity type — Run, Cycle, Swim, Row, Hike, Walk, Other. Required. Determines what fields are shown (e.g. Swim doesn't show pace).
  • Distance — km or miles, remembers preference. Required.
  • Duration — mm:ss format. Required.
  • Avg pace — calculated automatically from distance ÷ duration. Display only, not an input. auto-calculated
  • Strava / Apple Health sync — if a run was detected today, all three fields pre-fill. User can still edit. auto-synced
  • Effort — 4-face scale: Easy / Steady / Hard / Max. Feeds the AI recovery model. optional
  • Conditions — tap chips: Hot, Cold, Rainy, Windy, Treadmill, Trail, Race. Helps AI contextualise performance — e.g. "Your pace is 8% slower on hot days." optional
  • Notes — free text for route name, how the run felt, etc. optional
Pace is calculated, not entered. Never ask a runner to type their pace — they know their distance and time, not pace. We calculate it for them and display it. This is one of those small details that signals the app was built by people who actually run.
