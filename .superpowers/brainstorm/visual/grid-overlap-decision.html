Two decisions to make

Should Calories merge into Meal? Should Steps merge into Run/Cardio? Pick your preference for each — they are independent decisions.

Decision 1 — 🔥 Calories & 🍽️ Meal
🍽️

Merge — Calories lives inside Meal Recommended

Remove the standalone Calories tile. Calories becomes an optional field inside the Meal log. Users who want just a number get a quick preset row. Users who want full context get the description + feel fields too.

One tile, one intent — "I ate something"
Less clutter in the grid
Calories from MyFitnessPal auto-fill anyway
Slightly more steps for pure calorie-only users
🔥

Keep separate — Calories stays its own tile

Calories is a fast inline entry — one tap, type a number, done. Meal is the full description log. Two different speeds for two different users.

Ultra-fast for pure calorie trackers
Clear separation of intent
Two tiles for what feels like "food"
Confusing to new users — which one do I tap?
Decision 2 — 👟 Steps & 🏃 Run / Cardio
👟 🏃

Keep separate — Steps and Run stay their own tiles Recommended

Steps is a passive background number — most users never tap it because Apple Health / Fitbit auto-syncs it. Run is an intentional cardio session with distance, effort, and Strava. They are fundamentally different things.

Steps = passive metric, Run = active session
Runners get the right form without extra taps
Step-only users aren't overwhelmed by run fields
Two movement tiles in the grid
🏃

Merge — Steps folds into Run / Cardio

One "Movement" tile that opens a screen with two options: log a step count, or log a run/cardio session. Cleaner grid, one entry point for all movement.

Fewer tiles, cleaner grid
"Movement" is a coherent concept
Extra tap for step-only users
Steps is almost always auto-synced — adding a tap to something automatic feels like regression
